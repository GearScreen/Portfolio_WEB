ToyBox Warfare

Présentation du projet

Réaliser en groupe. Ce projet est un tower defence avec plusieurs types de tourelles que l'ont peut amélioré, le joueur peut également se transformer en tourelle,
il est limité par un nombre de munitions, les ennemis peuvent lâcher des munitions après leurs mort, il y a plusieurs types d'ennemis, plus on va loin dans
les vagues plus des ennemis puissants apparaîtront.

J'ai personnellement réaliser : système de tourelles, toutes les tourelles, placement des ponts, forme de tourelle du joueur.

Toutes les fonctionnalitée :

Toutes les tourelles ont des stats différentes : dégats, vitesse d'attaque, portée, coût.
Le joueur peut placer plusieurs types de tourelles : tourelle basique(tirs un projectile), canon(zone de dégats), tourelle à bulle(ralenti les ennemies),
la tourelle piège(place un piège sur la route), la boîte à musique(augmente les dégats des tourelles autour).
Le joueur peut aussi améliorer les tourelles, pour quelles gagne des stats et évolue graphiquement. Le joueur peut placer des ponts sur la route car elle bloque le joueur pendant les vagues

Technique :

J'ai créer une class de base pour toutes les tourelles qui permet de les améliorées.
Toutes les tourelles une fois placer rentre dans un état de construction ou elle ne peuvent rien faire, je peut définir leurs prochain état au passage à l'état de construction.

le background est une tilemap différente de la route se qui permet d'activé ou de désactivé le collider sur la tilemap de route pour empêcher le joueur de passer sur la route pendant les vagues.

Code :

Chaque tourelle hérite de plusieurs fonctions qu'elles doivent définir comme la fonction d'upgrade ce qui les rend uniques.

Post mortem :

Ce projet ma permis de beaucoup apprendre principalement en code, c'est un de mes premier projet qui utilise un singleton, c'est aussi le projet ou j'ai
appris à utiliser l'héritage (pour les tourelles principalement) et les state/stateMachine. j'ai aussi du utiliser animator pour la première fois.
